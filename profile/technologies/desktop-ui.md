# Desktop UI Technology Profile

## Discovery

Identify the user goal, primary workflow, platform conventions, existing visual language, UI framework, and input devices.

## Engineering Focus

- Define layout hierarchy, component ownership, state transitions, and event flow.
- Specify loading, empty, error, disabled, cancellation, and recovery states.
- Keep domain and automation logic out of UI state and prevent long work from freezing the interface.
- Define keyboard order, shortcuts, focus restoration, screen-reader labels, scaling, localization, resize, close, minimize, shutdown, and multi-window behavior.
- Prefer established components and design tokens.

## Validation

Map every interaction requirement to an observable test, including high DPI, font scaling, long text, keyboard access, cancellation, and error recovery.
