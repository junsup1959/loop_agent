# Implementation Task Profile

## Entry Conditions

- An assigned revision identifies one owner, target repository, base OID, branch, leased worktree, and bounded write scopes.
- Acceptance criteria, required validations, dependencies, and prohibited changes are explicit.
- The professional reference set is pinned before the activation starts.

## Execution Standard

- Inspect authoritative source evidence before editing.
- Preserve interfaces and compatibility unless the revision explicitly authorizes a versioned change.
- Keep implementation, generated outputs, and test artifacts within their declared roots.
- Validate a success path, a representative failure or boundary path, and affected integration edges.

## Completion Evidence

Return the exact head OID, changed paths, commands and exit results, artifact references, residual risks, and any request for additional context.
