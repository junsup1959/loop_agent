# Plan Delivery

Convert an approved scope into an outcome-based delivery plan with explicit dependency and risk control.

## Procedure

1. Identify deliverables and completion evidence.
2. Map hard dependencies, soft dependencies, and external constraints.
3. Assign organizational owners without embedding technical expertise into the role.
4. Find the critical path and safe parallel work.
5. Define milestone gates, retry limits, escalation triggers, and rollback points.
6. Record uncertainty that can change sequencing or completion confidence.

## Quality Rules

- Define milestones as outcomes, not activity lists.
- Give every dependency an owner and a completion signal.
- Avoid date certainty when dependency confidence is low.
- Keep optional improvements off the critical path.
- Include integration and validation time in the plan.

## Return Contract

Return:

- phased milestones and completion evidence;
- dependency and ownership map;
- critical path and parallel lanes;
- top risks and mitigations;
- retry, escalation, and rollback triggers;
- next coordination actions.

## Authority Boundary

This skill does not authorize scope changes, staffing decisions, approvals, model selection, tool permissions, or file edits. The assigned PM or PL retains those responsibilities.
