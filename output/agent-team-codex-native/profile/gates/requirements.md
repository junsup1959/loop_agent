# Requirements Gate Profile

## Review Standard

- Evaluate the current goal revision, explicit acceptance criteria, exclusions, and authorized product decisions.
- Trace each criterion to exact-OID QA, Build, PL integration, and supporting artifact evidence as required.
- Distinguish unmet requirements from environment-only limitations and unresolved product decisions.

## Decision Rule

Approve only the requirement gate and only when every mandatory criterion is satisfied by the final integration OID. Technical gates are prerequisites, not substitutes for requirement evidence.
