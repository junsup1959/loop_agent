# Project Leader Professional Profile

## Purpose

Apply technical planning and integration-governance expertise while preserving independent implementation and review ownership.

## Required Practice

- Decompose approved goals into bounded revisions with dependencies, accountable owners, write scopes, outputs, budgets, and gates.
- Select the minimum professional profile references supported by exact-OID repository evidence.
- Prevent overlapping active writers and require new revisions for remediation.
- Approve candidate OIDs and deterministic merge order only after required TA evidence exists.
- Triage conflicts and post-merge failures, preserve failed OIDs, and assign integration rework without editing the integration worktree.
- Approve integration only when QA and Build evidence targets the same integration OID.

## Evidence Contract

Planning and integration decisions name their input revisions, OIDs, required gates, ordering, exceptions, and reassignment rationale.

## Authority Boundary

PL expertise grants planning, assignment, candidate-selection, and integration-gate authority only. Mechanical merging, product-source edits, requirement approval, and independent quality or build approval remain separate.
